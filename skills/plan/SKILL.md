# 작업 계획 수립

## 입력

`$ARGUMENTS`로 작업 목표를 받습니다.

## 절차

### 1. 코드베이스 탐색

작업 목표와 관련된 파일과 아키텍처를 파악합니다:
- Glob/Grep으로 관련 파일 탐색
- 핵심 파일 읽기 (타입, 인터페이스, 라우팅 구조)
- 기존 패턴과 컨벤션 파악

### 2. 계획 수립

다음 항목을 포함한 계획을 작성합니다:

**수정 대상**: 변경이 필요한 파일 목록 (경로 포함)
**변경 내용**: 각 파일에서 무엇을 어떻게 변경할지
**테스트 전략**: 어떤 테스트가 필요한지 (단위/통합/E2E)
**예상 영향**: 다른 컴포넌트/모듈에 미치는 영향
**위험 요소**: 주의할 점, 잠재적 문제

### 3. 출력 형식

```markdown
## 계획: [작업 목표]

### 수정 대상
- `src/path/to/file.ts` — [변경 내용]
- ...

### 테스트 전략
- 단위 테스트: [대상]
- E2E 테스트: [시나리오]

### 예상 영향
- [영향 범위]

### 위험 요소
- [주의 사항]
```

## 주의사항

- 이 스킬은 **읽기 전용**입니다. 코드를 수정하지 않습니다.
- 추측 대신 실제 코드를 읽고 근거 있는 계획을 수립하세요.
- **모든 작업의 첫 번째 단계**로 사용됩니다.
